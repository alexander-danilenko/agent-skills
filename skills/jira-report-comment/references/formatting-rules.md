# Formatting Rules

- The comment is a delta to the ticket. Never repeat the title, description, or acceptance criteria. If the work matches the ticket exactly, the comment is one summary sentence and nothing else.
- Total length: 60-100 words. Most comments land near the lower end.
- Omit a section unless it carries information the ticket reader does not already have. Verify and Deploy default to absent.
- Use simple English - the team includes non-native speakers.
- Use only common IT terms (API, endpoint, database, deploy).
- No file paths, class names, or method names. No code blocks unless an exact value (env var, endpoint path, flag name) is essential.
- Present tense framed as outcomes ("Verification now checks the NPI Registry"), not activities ("Added NpiRegistryService").
- No em or en dashes - use regular hyphens (-) only.
- No special quotes - use regular "" or '' only.
- No semicolons or parenthetical asides.
- Sound like a teammate posting a quick update, not a formal report.
